수강신청 기반 취약점 학습 서비스 - API 목록
=============================================

목적
----
- 수강신청 사이트에 필요한 정상 기능 API와 OWASP Top 10:2025 취약점 노출 API를 함께 정리한다.
- 같은 API가 여러 취약점에 등장할 수 있다. 이 경우 API 자체가 중복이라는 뜻이 아니라, 같은 기능에서 서로 다른 취약점 시나리오를 관찰할 수 있다는 뜻이다.
- MCP는 별도 실습용 API가 아니라 실제 서비스 기능 API를 분석하는 흐름을 기본으로 한다.
- 수정·설정 변경과 삭제 요청은 DELETE/PATCH 대신 POST로 통일한다.
- **구현 참고:** `/api/debug/*`, `/api/system/*`, `/api/admin/audit-logs*`, `/api/admin/error-logs`, `/api/security/events` 는 현재 Nest 코드에서 **HTTP로 제공하지 않는다.**(DB·문서상 시나리오는 유지) 아래 OWASP 절에 남아 있는 해당 줄은 설계·학습용 목록이다.
- 각 API는 `role`을 함께 표기한다. 취약점 실습에서는 role 검증이 의도적으로 누락되거나 우회 가능하도록 구현할 수 있다.
- **팀원용 1페이지 요약:** `docs/vulnerability-matrix.md` (API × 취약점 유형, 구현 여부, 페이로드 예시)

Role 기준
---------
- `public`: 비로그인 사용자도 호출 가능
- `student`: 일반 수강신청 사용자
- `professor`: 강의/자료를 관리하는 교수 사용자
- `admin`: 관리자 사용자

응답 내 취약 정보 노출 예시
---------------------------
다음처럼 서버 내부 경로를 응답에 포함하는 방식은 취약점 분석용 웹에서 좋은 노출 포인트가 될 수 있다.

```json
{
  "fileId": 12,
  "filename": "secret.txt",
  "path": "/var/www/uploads/secret.txt"
}
```

활용 시나리오:
- A02 Security Misconfiguration: 서버 내부 디렉터리 구조 노출
- A04 Cryptographic Failures: 민감 파일 또는 secret 파일의 저장 위치 노출
- A10 Mishandling of Exceptional Conditions: 정상 기능 API의 에러 응답/파일 응답에 내부 경로 포함
- 파일 다운로드/업로드 API에서 Path Traversal 공격 힌트로 활용 가능


1. OWASP Top 10:2025 취약점 시나리오 API
======================================

(표기) 이 절에 나오는 `/api/debug/*`, `/api/system/*`, `/api/admin/audit-logs*`, `/api/admin/error-logs`, `/api/security/events` 등은 **현재 Nest에서 HTTP로 제공하지 않는다.** 교육용 시나리오·재구현 시 참고용이며, 실제 노출 API는 아래 `2. 서비스 정상 기능 API`와 상단 목적 절을 따른다.

[A01 Broken Access Control]
---------------------------
GET    /api/users/:userId                         role: student
- 다른 사용자의 프로필/민감 정보 조회(IDOR)

GET    /api/enrollments/:enrollmentId             role: student
- 다른 사용자의 수강신청 상세 조회(IDOR)

POST   /api/enrollments/:enrollmentId/cancel      role: student
- 다른 사용자의 수강신청 취소(IDOR)

GET    /api/admin                                   role: admin (문서 role, 실제 가드 없음)
- 일반 사용자의 관리자 대시보드 통계 조회

GET    /api/admin/users                           role: admin
- 일반 사용자의 관리자 사용자 목록 접근

POST   /api/admin/users/:userId/role              role: admin
- 일반 사용자의 권한 상승 또는 타 사용자 role 변경

GET    /api/files/:fileId/download                role: student, professor, admin
- 수강하지 않은 강의의 파일 다운로드(IDOR)


[A02 Security Misconfiguration]
-------------------------------
GET    /api/debug/configs                         role: admin
- 운영 설정/환경 설정 노출

GET    /api/admin/error-logs                      role: admin
- 상세 에러 로그 노출


[A03 Software Supply Chain Failures]
------------------------------------
GET    /api/system/dependencies                   role: admin
- 취약하거나 오래된 패키지 목록 노출

GET    /api/system/build-info                     role: admin
- 빌드 버전, 패키지 버전, 배포 정보 노출


[A04 Cryptographic Failures]
----------------------------
POST   /api/auth/login                            role: public
- 약한 토큰 서명 키, 과도하게 긴 토큰 만료 시간, 평문 비밀번호 비교 시나리오

POST   /api/auth/refresh                          role: public
- refresh token 재사용, 쿠키 보안 속성 미흡, 토큰 회전 실패

GET    /api/debug/secrets                         role: admin
- JWT secret, DB password, API key 등 민감 secret 노출

GET    /api/files/:fileId/download                role: student, professor, admin
- 만료되지 않는 다운로드 링크, 예측 가능한 파일 URL, 암호화되지 않은 민감 파일 다운로드
- 응답 또는 에러에 `"path": "/var/www/uploads/secret.txt"` 같은 내부 저장 경로를 포함할 수 있음


[A05 Injection]
---------------
(구현됨 — raw SQL 문자열 결합. 상세·페이로드는 `docs/vulnerability-matrix.md`, `docs/api-docs.txt` 각 API 절 참고)

POST   /api/auth/login                            role: public
- 로그인 사용자 조회 SQL Injection. `username`에 `admin' --`, `admin' OR '1'='1' --` 등으로 비밀번호 조건 우회

GET    /api/users/:userId                         role: student, professor, admin
- path `userId` SQL Injection (IDOR와 동시)

GET    /api/enrollments/me?userId=                role: student
- query `userId` SQL Injection · 타 사용자 목록 노출

GET    /api/enrollments/:enrollmentId             role: student
- path `enrollmentId` SQL Injection (IDOR와 동시)

POST   /api/enrollments                           role: student
- body `userId` / `courseId` / `status` SQL Injection (INSERT)

POST   /api/enrollments/:enrollmentId/cancel      role: student
- path `enrollmentId` SQL Injection (UPDATE)

GET    /api/courses?keyword=                      role: student, professor, admin
- 강의 검색 `keyword` SQL Injection (`' OR '1'='1` 등). 단독으로는 공개 10건과 비슷해 보일 수 있음

GET    /api/courses?category=                     role: student, professor, admin
- `category` SQL Injection · `is_visible=false` 조건 우회로 숨김 강의 포함 15건 반환 가능

GET    /api/courses/:courseId                     role: student, professor, admin
- path `courseId` SQL Injection

POST   /api/courses/:courseId/visibility          role: admin (문서 role, 실제 가드 없음)
- path `courseId` · body `isVisible` SQL Injection

GET    /api/courses/:courseId/files               role: student, professor, admin
- path `courseId` SQL Injection

GET    /api/files/:fileId/download                role: student, professor, admin
- path `fileId` SQL Injection (IDOR와 동시)

POST   /api/files/:fileId/delete                  role: professor, admin
- path `fileId` SQL Injection

POST   /api/admin/users/:userId/role              role: admin (문서 role, 실제 가드 없음)
- path `userId` · body `role` SQL Injection · 권한 상승과 동시

(미구현 — 설계·학습용만, HTTP 없음)
POST   /api/auth/register                         role: public
- 문서상 회원가입 SQLi 시나리오는 있으나 **현재는 TypeORM** → SQLi 없음. `role=admin` body는 A07(인증/권한) 실습

GET    /api/admin/audit-logs?path=                role: admin  (미구현)
POST   /api/debug/query-test                      role: admin  (미구현)


[A06 Insecure Design]
---------------------
POST   /api/enrollments                           role: student
- 프론트 버튼 비활성화 우회, 서버 측 정원 검증 부재, 정원 초과 신청, 중복 신청, 신청 기간 미검증

POST   /api/courses/:courseId/visibility          role: admin (가드 없음 — student도 호출 가능)
- 숨겨진 강의 공개 상태 조작


[A07 Authentication Failures]
-----------------------------
POST   /api/auth/register                         role: public
- 비밀번호 복잡도/길이 정책 없음. body `role=admin` 등 **권한 상승 가입** (SQLi는 미구현)

POST   /api/auth/login                            role: public
- 로그인 시도 제한 없음, 계정 잠금 없음

POST   /api/auth/refresh                          role: public
- 만료/폐기된 refresh token 재사용

POST   /api/auth/logout                           role: student, professor, admin
- refresh token 폐기 실패

GET    /api/auth/me                               role: student, professor, admin
- 잘못된 토큰 처리, 만료 토큰 허용


[A08 Software or Data Integrity Failures]
-----------------------------------------
POST   /api/files/:fileId/delete                  role: professor, admin
- role 조작으로 얻은 professor/admin 권한을 이용한 목데이터 강의 자료 삭제, 일정 시간 후 자동 복구/무결성 보장 확인

POST   /api/debug/configs/:key                    role: admin
- 런타임 설정 변조, 보안 옵션 비활성화


[A09 Security Logging and Alerting Failures]
--------------------------------------------
GET    /api/admin/audit-logs                      role: admin
- 감사 로그 조회

GET    /api/admin/error-logs                      role: admin
- 에러 로그 조회

POST   /api/admin/audit-logs/clear                role: admin
- 공격 흔적 삭제

GET    /api/security/events                       role: admin
- 보안 이벤트 누락/불완전 기록 확인


[A10 Mishandling of Exceptional Conditions]
-------------------------------------------
GET    /api/courses/:courseId                     role: student, professor, admin
- 존재하지 않거나 형식이 잘못된 courseId 처리 실패, stack trace 노출

POST   /api/enrollments                           role: student
- 정원 초과/중복 신청/동시 요청 충돌 시 예외 처리 실패, fail-open 또는 raw error 노출

GET    /api/files/:fileId/download                role: student, professor, admin
- 존재하지 않는 파일 또는 권한 없는 파일 요청 시 내부 경로/stack trace 노출

POST   /api/courses/:courseId/visibility          role: admin
- 공개/숨김 변경 요청에서 누락/잘못된 필드 처리 실패, validation 예외가 내부 구조를 포함해 반환됨


2. 서비스 정상 기능 API
======================

[Auth]
------
POST /api/auth/register                           role: public
- 회원가입

POST /api/auth/login                              role: public
- 로그인 및 JWT access token 발급

POST /api/auth/refresh                            role: public
- refresh token 기반 access token 재발급

POST /api/auth/logout                             role: student, professor, admin
- refresh token 폐기

GET  /api/auth/me                                 role: student, professor, admin
- 현재 로그인 사용자 조회


[Users]
-------
GET /api/users/:userId                            role: student, professor, admin
- 사용자 상세 조회. 마이페이지와 IDOR 실습에서 사용


[Courses]
---------
GET    /api/courses                               role: student, professor, admin
- 강의 목록, 검색, 카테고리 필터

GET    /api/courses/:courseId                     role: student, professor, admin
- 강의 상세

POST   /api/courses/:courseId/visibility          role: admin
- 시드/목데이터 강의의 공개/숨김 상태 변경. 관리자 기능

[Enrollments]
-------------
GET    /api/enrollments/me                        role: student
- 내 수강신청 목록

GET    /api/enrollments/:enrollmentId             role: student
- 수강신청 상세

POST   /api/enrollments                           role: student
- 단건 수강신청

POST   /api/enrollments/:enrollmentId/cancel      role: student
- 수강신청 취소


[Files]
-------
GET    /api/courses/:courseId/files               role: student, professor, admin
- 강의 자료 목록

GET    /api/files/:fileId/download                role: student, professor, admin
- 강의 자료 다운로드
- 응답에 내부 저장 경로(`path`)를 포함하면 정보 노출/Path Traversal 실습 포인트로 사용할 수 있음

POST   /api/files/:fileId/delete                  role: professor, admin
- 목데이터 강의 자료 삭제. role 조작으로 professor/admin 진입 시 사용하고 일정 시간 후 복구 여부 확인


[Admin]
-------
GET   /api/admin                              role: admin (가드 없음)
- 관리자 대시보드 통계 (하드코딩)

GET   /api/admin/users                            role: admin
- 사용자 목록

POST /api/admin/users/:userId/role               role: admin
- 사용자 권한 변경

(아래 감사·에러 로그 HTTP API는 **현재 Nest 구현에 없음**. DB·OWASP 시나리오·`docs/api-docs.txt` [구현 제거 안내]/[Logs] 참고)
- GET   /api/admin/audit-logs          (미구현)
- POST  /api/admin/audit-logs/clear   (미구현)
- GET   /api/admin/error-logs         (미구현)

[System / Debug] (미구현)
-------------------------
아래 전부 **현재 Nest 구현에 없음**. 설계·학습용 시나리오는 OWASP 절 및 `api-docs.txt` 참고.

- GET   /api/system/dependencies      (미구현)
- GET   /api/system/build-info        (미구현)
- GET   /api/debug/configs            (미구현)
- POST  /api/debug/configs/:key       (미구현)
- GET   /api/debug/secrets            (미구현)
- POST  /api/debug/query-test         (미구현)
- GET   /api/security/events          role: admin  (미구현) — 보안 이벤트 통합 조회, 설계만 유지
