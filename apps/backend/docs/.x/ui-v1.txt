수강신청 기반 취약점 학습 서비스 - 화면 구성 및 취약점 설계 문서
==================================================================

목적
----
본 문서는 수강신청 기반 웹 애플리케이션에서 필요한 화면(UI) 구성과,
각 화면에서 의도적으로 노출시킬 취약점을 OWASP Top 10:2025 기준으로 정리한 문서이다.

설계 원칙
---------
- 실제 서비스처럼 보이도록 구성한다 (Juice Shop 방식)
- 취약점은 별도 lab이 아닌 기능 안에 자연스럽게 삽입한다
- MCP 취약점 분석 툴이 API 기반으로 탐지 가능하도록 한다
- MCP는 별도 실습 API가 아니라 실제 서비스 기능 API를 분석하는 흐름을 기본으로 한다
- debug/system API는 운영 중 잘못 노출된 관리자/진단 API로만 취급한다
- 각 화면은 가능한 한 실제 수강신청 흐름을 따른다


1. 로그인 / 회원가입 페이지
==========================

경로:
- /login
- /register

기능:
- 회원가입
- 로그인
- JWT 발급
- refresh token 발급/재발급
- 로그아웃

취약점:
- 비밀번호 복잡도/길이 정책 없음
- 로그인 시도 제한 없음
- JWT/토큰 취약 구성
- refresh token 재사용/폐기 실패

연결 API:
- POST /api/auth/register
- POST /api/auth/login
- POST /api/auth/refresh
- POST /api/auth/logout
- GET  /api/auth/me


2. 대시보드 (마이페이지)
========================

경로:
- /dashboard

기능:
- 내 정보 조회
- 내 수강 목록
- 수강신청 상세 확인
- 수강 취소

취약점:
- 다른 userId 조회 가능 (IDOR)
- 다른 enrollmentId 조회 가능 (IDOR)
- 다른 사용자의 수강신청 취소 가능
- 민감 정보 노출

연결 API:
- GET    /api/auth/me
- GET    /api/users/:userId
- GET    /api/enrollments/me
- GET    /api/enrollments/:enrollmentId
- POST   /api/enrollments/:enrollmentId/cancel


3. 강의 목록 페이지
===================

경로:
- /courses

기능:
- 강의 검색
- 필터링
- 숨김/비공개 강의 노출 여부 확인
- 강의 상세 이동

취약점:
- SQL Injection (검색)
- SQL Injection 또는 조건 우회 (카테고리 필터)
- 숨겨진 강의 노출

연결 API:
- GET /api/courses
- GET /api/courses?keyword=
- GET /api/courses?category=


4. 강의 상세 페이지
==================

경로:
- /course/:courseId

기능:
- 강의 정보
- 수강신청
- 수강 취소
- 강의 자료 목록 확인

취약점:
- 프론트 신청 버튼 비활성화 우회
- 정원 초과 신청 (Race Condition)
- 중복 신청
- 신청 기간/정원 정책 우회
- IDOR
- 에러 메시지 노출

연결 API:
- GET    /api/courses/:courseId
- POST   /api/enrollments
- POST   /api/enrollments/:enrollmentId/cancel
- GET    /api/courses/:courseId/files


5. 강의 자료 (파일 영역)
=======================

경로:
- /course/:courseId/files
- /admin/files 또는 강의 관리 상세 내부

기능:
- 파일 목록 조회
- 파일 다운로드
- 파일 삭제

취약점:
- IDOR (다른 파일 접근)
- Path Traversal
- 민감 파일 노출
- role 조작 후 목데이터 강의 자료 삭제
- 삭제 후 일정 시간 내 목데이터 복구 가능 여부 확인
- 만료되지 않는 다운로드 링크
- 파일/에러 응답의 내부 서버 경로 노출

연결 API:
- GET    /api/courses/:courseId/files
- GET    /api/files/:fileId/download
- POST   /api/files/:fileId/delete


6. 관리자 페이지
===============

경로:
- /admin

기능:
- 유저 관리
- 강의 관리
- 로그 조회
- 시스템/보안 설정 확인

취약점:
- 관리자 API 접근 우회
- 일반 사용자의 role 변경
- 설정 노출
- 로그 삭제

연결 API:
- GET   /api/admin
- GET   /api/admin/users
- POST /api/admin/users/:userId/role
- GET   /api/admin/audit-logs
- POST  /api/admin/audit-logs/clear
- GET   /api/admin/error-logs
- GET   /api/system/dependencies
- GET   /api/system/build-info


7. 관리자 - 강의 관리 상세
==========================

경로:
- /admin/courses
- /admin/courses/:courseId

기능:
- 시드/목데이터 강의 조회
- 강의 공개/숨김 상태 변경
- 강의 자료 관리로 이동

취약점:
- 관리자/교수 권한 우회
- 숨겨진 강의 공개 상태 조작
- 잘못된 소유권 검증

연결 API:
- GET    /api/courses
- GET    /api/courses/:courseId
- POST /api/courses/:courseId/visibility


8. 관리자 - 로그 / 보안 이벤트 페이지
====================================

경로:
- /admin/logs
- /admin/security/events

기능:
- 감사 로그 조회
- 에러 로그 조회
- 보안 이벤트 조회
- 로그 삭제
- 로그 검색/필터링

취약점:
- 공격 흔적 삭제
- 보안 이벤트 누락
- 에러 로그의 민감 정보 노출
- 로그 검색 조건 injection
- stack trace, 내부 경로, DB 에러 메시지 노출

연결 API:
- GET  /api/admin/audit-logs
- GET  /api/admin/audit-logs?path=
- POST /api/admin/audit-logs/clear
- GET  /api/admin/error-logs
- GET  /api/security/events


9. 관리자 - 시스템 / 보안 설정 페이지
====================================

경로:
- /admin/system
- /admin/security/settings

기능:
- dependency 목록 확인
- build 정보 확인
- debug config 확인/수정

취약점:
- 운영 설정 노출
- dependency/build 정보 노출
- 런타임 보안 설정 변조

연결 API:
- GET   /api/system/dependencies
- GET   /api/system/build-info
- GET   /api/debug/configs
- POST /api/debug/configs/:key


10. A10 예외 처리 취약점 탐지 흐름
================================

화면 노출:
- 별도 Crash/Exception 실습 화면은 만들지 않는다
- 수강신청, 강의 상세, 파일 같은 정상 기능에서 비정상 입력을 보냈을 때 자연스럽게 드러나도록 한다

기능:
- 존재하지 않는 강의/파일 조회
- 정원 초과, 중복 신청, 동시 신청 요청
- 누락되거나 형식이 잘못된 요청 필드 전송

취약점:
- stack trace 노출
- DB 에러 메시지 노출
- timeout/fail-open 처리 실패
- 내부 서버 경로 노출
- validation/parser 에러가 내부 구조를 포함해 반환됨

연결 API:
- GET   /api/courses/:courseId
- POST  /api/enrollments
- GET   /api/files/:fileId/download


1차 구현 우선순위
================

필수 화면:
- 로그인 / 회원가입
- 대시보드
- 강의 목록
- 강의 상세
- 강의 자료 영역
- 관리자 페이지

관리자 상세 화면:
- 강의 관리 상세
- 로그 / 보안 이벤트
- 시스템 / 보안 설정

선택 화면:
- A10은 별도 화면 없이 기존 기능 API의 예외 응답에서 탐지되도록 둔다.


결론
====
파일 영역은 핵심 공격 포인트이며,
수강신청/강의관리/관리자 기능 안에 취약점을 자연스럽게 포함한다.
debug/system 계열 API는 관리자/진단 기능이 잘못 노출된 형태로만 사용하고,
A10은 별도 crash API가 아니라 기존 기능의 예외 처리 실패로 드러나게 설계한다.
