아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - .cursor/rules/01-common.mdc

==============================================

구현될 컴포넌트 위치) src/components/dashboard/dashboard.tsx

조건-API) 계정 정보 조회 엔드포인트 경로: GET /api/auth/me
조건-API) 수강신청 내역 조회 엔드포인트 경로: GET /api/enrollments/me
조건-API) 수강신청 취소 엔드포인트 경로: POST /api/enrollments/:enrollmentId/cancel

핵심요구사항) dashboard 페이지에 목데이터들을 제거하고, 계정 정보 조회 엔드포인트와 수강신청 내역 조회 엔드포인트를 호출하여 데이터를 표시할 것.
            - 계정 정보 중, 다음의 값들은 하드코딩 된 값을 사용할 것.
              - user.grade: 3학년
              - user.advisor: 리누스 토발즈
              - user.major: 컴퓨터공학
            - 계정 정보 중, 다음의 값들을 API 의 응답에서 가져올 것.
              - user.username: response.username
              - user.id: response.id 를 다음과 같이 조합할 것.
                - 결과: 2610101
                  - 26: 년도 (하드코딩)
                  - 101: 무의미한 숫자 (하드코딩)
                  - 01: 학번 마지막 두자리 (API 응답에서 가져온 response.id, id가 1이라면 01, id가 2라면 02, ...)
            - 수강신청 정보중 다음의 값들을 API 의 응답에서 가져올 것.
              - enrollments.courseTitle: response.courseTitle
              - enrollments.professor: response.professorName
              - enrollments.enrolledAt: response.enrolledAt
              - enrollments.cousrseTitle: response.courseTitle
              - enrollments.courseId: response.courseId
              - enrollments.status: response.status
                - status가 enrolled 이면 수강신청 취소 버튼을 표시할 것.
                - status가 dropped 이면 렌더링 하지 않을 것.

            - 수강신청 취소 버튼을 클릭하면, 수강신청 취소 엔드포인트를 호출하여 수강신청을 취소할 것.
