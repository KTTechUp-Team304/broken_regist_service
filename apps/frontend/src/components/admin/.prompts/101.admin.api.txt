            - .cursor/rules/01-common.mdc

==============================================

조건-파일경로) 구현될 TSX 파일경로: src/components/admin/admin.tsx
조건-파일경로) 구현될 TSX 파일경로: src/components/admin/admin-courses.tsx
조건-파일경로) 구현될 TSX 파일경로: src/components/admin/admin-users.tsx

조건-파일경로) 구현될 API 파일경로: src/commons/api/admin-api.ts

조건-엔드포인트) 강의 목록 조회 엔드포인트 경로: GET /api/courses
조건-엔드포인트) 사용자 목록 조회 엔드포인트 경로: GET /api/admin/users
조건-엔드포인트) 사용자 권한 변경 엔드포인트 경로: POST /api/admin/users/:userId/role

핵심요구사항) /admin/courses, /admin/users 페이지에 목데이터들을 제거하고, 각 페이지에 필요한 API를 호출하여 데이터를 표시할 것.
          - /admin/courses
            - 강의 목록 조회 엔드포인트를 호출하여 다음의 값들을 매핑할것.
              - course.courseTitle: response.courseTitle
              - course.professorName: response.professorName
              - course.category: response.category
              - course.isVisible: response.isVisible
              - course.currentCount: response.currentCount
              - course.maxCapacity: response.maxCapacity
              - course.createdAt: response.createdAt
            
          - /admin/users
            - 사용자 목록 조회 엔드포인트를 호출하여 다음의 값들을 매핑할것.
              - user.username: response.username
              - user.role: response.role
              - user.createdAt: response.createdAt
              - user.lastLogin: response.recentLoginDate
              - user.status: response.status

핵심요구사항) 사용자 권한 변경 버튼을 클릭하면, 사용자 권한 변경 엔드포인트를 호출하여 사용자 권한을 변경할 것.
              - 엔드포인트: POST /api/admin/users/:userId/role
                  - 파라미터: userId: 사용자 ID
                  - 요청 바디: { role: 'student' | 'professor' | 'admin' }

핵심요구사항) 모든 날짜 형식은 YYYY-MM-DD 형식으로 표시할 것.
