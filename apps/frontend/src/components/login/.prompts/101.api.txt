아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - .cursor/rules/01-common.mdc

==============================================

조건-파일경로) 구현될 TSX 파일경로: src/components/login/login.tsx
조건-파일경로) 구현될 Provider 파일경로: src/context/authContext.tsx

조건-API) 로그인 엔드포인트 경로: POST /api/auth/login

핵심요구사항) 로그인 페이지에 목데이터들을 제거하고, 로그인 엔드포인트를 호출하여, 로그인 기능을 구현할 것.
            - 로그인 성공 시, authContext에 저장된 user 정보를 초기화하고, /dashboard 페이지로 이동할 것.
            - 로그인 성공 시, 로그인 정보를 localStorage에 저장할 것.
              - key: accessToken
              - value: 로그인 엔드포인트 응답에서 가져온 accessToken
              - key: refreshToken
              - value: 로그인 엔드포인트 응답에서 가져온 refreshToken
            - 로그인 실패 시, 에러 메시지를 표시할 것.

            - 로그아웃 버튼을 클릭하면, authContext에 저장된 user 정보를 초기화하고, /login 페이지로 이동할 것.
              - 로그아웃 시, localStorage에 저장된 accessToken을 제거할 것.

핵심요구사항) authContext.tsx 파일의 위치를 src/context/authContext.tsx 에서 src/commons/providers/auth-context/auth-context.tsx 로 이동시킬 것.
핵심요구사항) 현재 파일에 구현되어있는 localStorage 의 key, value 명을 현재 프롬프트에 맞게 수정할 것.
